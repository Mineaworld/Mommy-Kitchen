# Documentation Index

This folder is the single source of truth for product, architecture, and execution.

## Structure

- `docs/product/PRD.md`: Product requirements and acceptance criteria
- `docs/architecture/ARCHITECTURE.md`: System design and runtime behavior
- `docs/architecture/ERD.md`: Data model and table definitions
- `docs/architecture/DFD.md`: Data flow diagrams for key user and admin paths
- `docs/architecture/API_SPEC.md`: API and server action contracts
- `docs/agile/BACKLOG.md`: Epics, stories, priorities, points, and dependencies
- `docs/agile/SPRINT_0.md`: Setup and foundation sprint plan
- `docs/agile/SPRINT_1.md`: First delivery sprint plan
- `docs/agile/DEFINITION_OF_READY.md`: Preconditions before work starts
- `docs/agile/DEFINITION_OF_DONE.md`: Quality gates before completion
- `docs/agile/WORKFLOW.md`: Solo agile cadence and ceremonies

## Decision Defaults

- Documentation format: Markdown + Mermaid
- Tracking: In-repo markdown backlog
- Cadence: 1-week sprints
- Prioritization: MoSCoW
- Estimation: Story points `1, 2, 3, 5, 8`
- Language: English primary with Khmer examples where useful
