# Definition of Ready (DoR)

A story is ready when all criteria below are true:

1. Story format is clear: user, goal, outcome.
2. Acceptance criteria are testable and explicit.
3. Priority (MoSCoW) is set.
4. Story points are assigned (`1,2,3,5,8`).
5. Dependencies are listed.
6. Required design or API references are linked.
7. Security and accessibility considerations are noted.
8. Scope fits one sprint or is split into smaller stories.
