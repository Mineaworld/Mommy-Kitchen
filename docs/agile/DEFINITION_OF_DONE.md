# Definition of Done (DoD)

A story is done when all criteria below are true:

1. Acceptance criteria pass in local verification.
2. Code is typed and lint/build checks pass.
3. No regression in existing flows.
4. Security checks for relevant endpoints are covered.
5. Accessibility checks are completed for UI changes.
6. Documentation is updated (`PRD`, API, or architecture if changed).
7. Backlog item status and notes are updated.
8. Work is reviewed against sprint goal.
