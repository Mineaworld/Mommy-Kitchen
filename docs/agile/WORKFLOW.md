# Solo Agile Workflow

## Cadence

- Sprint length: 1 week
- Planning day: Monday
- Review and retrospective: Friday

## Weekly Rituals

1. Plan sprint scope from `BACKLOG.md`.
2. Confirm each selected story meets DoR.
3. Execute development and update progress daily.
4. Validate completed stories against DoD.
5. Run sprint review and note carry-over items.
6. Capture retrospective actions for next sprint.

## Prioritization and Commitment

- Prioritization model: MoSCoW
- Commit `Must` first, then highest-value `Should`.
- Reserve 20% capacity for defects, docs, and refactor.

## Story Lifecycle

- `Proposed` -> `Ready` -> `In Progress` -> `Review` -> `Done`

## Estimation

- Story points: `1, 2, 3, 5, 8`
- Re-estimate only when scope changes materially.

## Solo Review Loop

- Before closing story:
  - Re-read acceptance criteria
  - Re-test critical path
  - Update docs and backlog notes
